# 🎯 90分アイディアソン ワークショップ手順書

このワークショップでは、Amazon Q Developerとの壁打ちで90分でアイディアを膨らませ、モックを作成し、チームビルディングまで行います。

## 📋 全体の流れ（90分）

```
0-5分   ► イントロ・環境確認
5-35分  ► アイディア膨らまし（Step1→2→3）
35-60分 ► モック作成（Step4）
60-65分 ► スライド作成（Step6）
65-70分 ► 提出（Step7）
70-85分 ► 共有・チームビルディング
85-90分 ► Next Step
```

---

## 🔧 事前準備（開始前に完了させる）

### 1. 環境確認

以下が動作することを確認してください：

```bash
# Python 3.11の確認
python --version
# Python 3.11.x を期待

# uvの確認
uv --version
```

**Amazon Q Developer:**
- VSCode拡張機能がインストールされていることを確認
- VSCodeでAmazon Qチャットが開けることを確認
- （オプション）CLI版を使いたい場合は `q --version` で確認

### 2. リポジトリのクローン

```bash
git clone https://github.com/ist-whitebox/ai-hackathon-2025-ideathon.git
cd ai-hackathon-2025-ideathon
```

### 3. .envファイルの設定

```bash
cd your_work/
cp .env.sample .env
```

`.env`ファイルを編集し、運営から配布されたAPIキーを設定：

```bash
AWS_BEARER_TOKEN_BEDROCK=your-api-key-here
AWS_REGION=ap-northeast-1
AWS_BEDROCK_MODEL_ID=global.anthropic.claude-haiku-4-5-20251001-v1:0
```

### 4. 身近な課題を考える

アイディアの種となる「身近な課題」を1つ考えておいてください（メモ程度でOK）。

例：
- 「毎日の献立を考えるのが面倒」
- 「会議の議事録作成に時間がかかる」
- 「英語のメールを書くのが苦手」

## Step 1: 顧客理解 🤖

**目的**: 身近な課題から顧客像を作り、行動を分析し、改善すべき課題を特定する

### 生成フェーズ

1. **VSCodeで正しいフォルダを開く**

```

```

2. **Amazon Q Developerチャットを開く**

- VSCode左側のAmazon Qアイコンをクリック

`your_work/prompts/step1_customer_understanding.md` を開き、プロンプトをコピーします。

`[ここに自分の課題を記入]` の部分を、事前に考えた課題に置き換えてください。


`your_work/ideation/step1_persona.md` を開いて内容を確認し、以下の観点で修正してください：

**必要に応じてファイルを直接編集してください。AIの出力をそのまま受け入れず、自分の頭で考えることが重要です。**

## Step 4: モック作成 🤖

**目的**: PR/FAQからStrands Agentsを活用した動くモックを作成する

### VSCodeフォルダの変更（重要！）

**新しいVSCodeウィンドウで `your_work/` フォルダを開く:**

```
File > Open Folder → ai-hackathon-2025-ideathon/your_work/ を選択
```

### モック作成の流れ

1. **Amazon Q Developerチャットを開く**

- VSCode左側のAmazon Qアイコンをクリック
- またはショートカット: `Cmd+Shift+P` (Mac) / `Ctrl+Shift+P` (Win) → 「Amazon Q: Open Chat」

2. **モック作成を依頼**

以下のように入力してください：

```
@ideation/step3_prfaq.md を参照して、モックアプリケーションを作成してください。
```

3. **技術スタックを選択**

Amazon Qから技術スタックの選択を求められます：

- **オプション1: Streamlit + Strands Agents（推奨）**
  - 最速で実装可能
  - インタラクティブなUIが簡単
  - 90分のアイディアソンに最適

- **オプション2: FastAPI + HTML + Strands Agents**
  - デザインの自由度が高い
  - コード量が多い

**初心者はStreamlitを選択することを推奨します。**

4. **モックが自動生成される**

`mock/` ディレクトリにアプリケーションが生成されます。

5. **.envファイルをコピー**

```bash
cp .env mock/.env
```

6. **仮想環境を作成して起動**

```bash
cd mock/

# 仮想環境作成
uv venv --python 3.11

# 仮想環境を有効化
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# 依存パッケージをインストール
uv pip install -r requirements.txt

# アプリを起動
streamlit run app.py  # Streamlitの場合
# または
python app.py         # FastAPIの場合
```

7. **ブラウザで開く**

- Streamlit: http://localhost:8501
- FastAPI: http://localhost:8000

### モック完成後の確認（必須）

8. **チェックリストで検証**

Amazon Q Developerチャットに以下のように依頼してください：

```
チェックリストに沿ってチェックしてください
```

9. **チェック結果を確認**

- ✅ Phase 0完了（依存関係）
- ✅ Phase 1完了（Strands Agents記法）
- ✅ Phase 2完了（起動確認）

10. **問題があれば修正**

チェックで問題が見つかった場合は、Amazon Qチャットで修正を依頼してください。

**💡 CLI版を使う場合（オプション）:**
```bash
cd your_work/
q chat
# 「チェックリストに沿ってチェックしてください」と入力
```

## ⚠️ トラブルシューティング

### Amazon Q Developer拡張機能が動かない

1. **拡張機能のインストール確認**
   - VSCode左側の拡張機能アイコンをクリック
   - 「Amazon Q」で検索
   - インストールされているか確認

2. **再起動**
   - VSCodeを再起動

3. **ログイン確認**
   - Amazon Qアイコンをクリック
   - ログイン状態を確認

**CLI版を使う場合（オプション）:**
```bash
# インストール確認
q --version

# 再インストール
npm install -g @aws/amazon-q-developer-cli
```

### モック作成が失敗する

- `ideation/step3_prfaq.md` が保存されているか確認
- PR/FAQの内容が具体的か確認
- エラーメッセージをAmazon Q Developerチャットに貼り付けて相談

### アプリが起動しない

1. **Python バージョンを確認**

```bash
python --version  # Python 3.11.x であることを確認
```

2. **.envファイルが正しくコピーされているか確認**

```bash
ls -la mock/.env
cat mock/.env  # API キーが設定されているか確認
```

3. **依存パッケージを再インストール**

```bash
cd mock/
rm -rf .venv
uv venv --python 3.11
source .venv/bin/activate
uv pip install -r requirements.txt
```

### Strands Agentsのエラーが出る

1. **.envの設定を確認**

```bash
# 以下の変数が設定されているか確認
AWS_BEARER_TOKEN_BEDROCK=your-api-key-here
AWS_REGION=ap-northeast-1
AWS_BEDROCK_MODEL_ID=global.anthropic.claude-haiku-4-5-20251001-v1:0
```

2. **strands-agentsパッケージがインストールされているか確認**

```bash
pip list | grep strands
```

### 画面が表示されない

1. **ポートが使用中でないか確認**

```bash
# Streamlit の場合
lsof -i :8501

# FastAPI の場合
lsof -i :8000
```

2. **ブラウザのキャッシュをクリア**

3. **別のブラウザで試す**

### 困ったら
