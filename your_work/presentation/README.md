# プレゼンテーション資料 自動生成

このフォルダには、アイディアソンの成果物から自動的にプレゼン資料を生成するツールが含まれています。

## 📁 ファイル構成

```
presentation/
├── README.md           # このファイル
├── generate.py         # プレゼン生成スクリプト
├── requirements.txt    # 依存パッケージ
└── index.html          # 生成されたプレゼン資料（実行後）
```

## 🎯 使い方

### 前提条件

以下が完了していることを確認してください：

- ✅ Step1-3の成果物が保存されている
  - `ideation/step1_persona.md`
  - `ideation/step2_ideas.md`
  - `ideation/step3_prfaq.md`
- ✅ モック作成が完了している（`mock/`フォルダ）
- ✅ `.env`ファイルが設定されている

### 実行手順

1. **モックの仮想環境を有効化**

```bash
cd your_work/mock/
source .venv/bin/activate  # Windows: .venv\Scripts\activate
```

2. **プレゼン生成スクリプトを実行**

```bash
cd ..
python presentation/generate.py
```

3. **デザインを選択**

4つの質問に答えて、プレゼンのデザインを選択します：

```
🎨 デザインを選択してください
============================================================

[1/4] カラーテーマを選んでください:
  1. パープル・グラデーション (知的・落ち着き) [デフォルト]
  2. ブルー・グリーン (爽やか・テック系)
  3. オレンジ・ピンク (温かみ・消費者向け)
  4. ダーク・モード (クール・スタイリッシュ)
選択 (1-4, Enter=1): 

[2/4] 雰囲気を選んでください:
  1. ビジネス・フォーマル [デフォルト]
  2. フレンドリー・カジュアル
  3. テック・イノベーション
選択 (1-3, Enter=1): 

[3/4] アニメーションを選んでください:
  1. 控えめ
  2. 標準 [デフォルト]
  3. ダイナミック
選択 (1-3, Enter=2): 

[4/4] その他、こだわりたいポイントがあれば教えてください:
  例: 「ポップな印象にしたい」「文字を大きめに」「シンプルに」
  ※ 改行なしで1行で入力してください。なければEnterでスキップ
要望: 

✅ 設定完了！プレゼンを生成中...
```

💡 **Tips**: 急いでいる場合は、すべてEnterキーを押すだけでデフォルト設定で生成できます。

4. **生成完了を確認**

```
🎉 プレゼン資料の作成が完了しました！
📍 ファイルの場所: presentation/index.html
```

5. **ブラウザで開く**

`presentation/index.html` をダブルクリックするか、ブラウザで開きます。

## 🎨 プレゼンの特徴

### デザインテーマ（選択可能）

#### カラーテーマ
1. **パープル・グラデーション** - 知的で落ち着いた印象、ビジネス向け
2. **ブルー・グリーン** - 爽やかでテック系、環境系に合う
3. **オレンジ・ピンク** - 温かみがあり消費者向け、エンタメ系に合う
4. **ダーク・モード** - クールでスタイリッシュ、開発者向け

#### 雰囲気
1. **ビジネス・フォーマル** - 企業向けサービス、B2B向け
2. **フレンドリー・カジュアル** - 消費者向け、B2C向け
3. **テック・イノベーション** - スタートアップ、技術系向け

#### アニメーション
1. **控えめ** - フェードインのみ、落ち着いた動き
2. **標準** - バランスの取れた動き
3. **ダイナミック** - インパクトのある動き

### デザイン

- **LP風のスクロール型デザイン**
- モダンでビジュアルが美しい
- レスポンシブデザイン（PC・タブレット・スマホ対応）
- スムーズなアニメーション
- **選択したテーマに基づいて色やトーンが自動調整**

### 構成（6セクション）

1. **タイトル** - サービス名とキャッチコピー
2. **課題** - 誰のどんな課題を解決するか
3. **解決策** - サービス概要と主要機能3つ
4. **デモ** - モックの使い方（ステップ形式）
5. **強み** - 独自の強み3つ
6. **チーム募集** - メッセージと実現したい世界

### 発表方法

- ブラウザでフルスクリーンモード（F11キー）にして発表
- スクロールしながらセクションごとに説明
- 発表時間: 3分を想定

## ✏️ カスタマイズ

### 内容の調整

AIが生成した内容が気に入らない場合、以下の方法で調整できます：

1. **成果物を修正して再生成**

```bash
# ideation/配下のファイルを編集
vi ideation/step3_prfaq.md

# 再実行
python presentation/generate.py
```

2. **HTMLを直接編集**

```bash
# 生成されたHTMLを直接編集
vi presentation/index.html
```

### デザインの変更

`index.html` の `<style>` タグ内のCSSを編集することで、以下をカスタマイズできます：

- **色**: `background`, `color`, `border-color`
- **フォント**: `font-family`, `font-size`
- **レイアウト**: `padding`, `margin`, `grid-template-columns`
- **アニメーション**: `animation`, `transition`

## ⚠️ トラブルシューティング

### エラー: 成果物ファイルが不足しています

**原因**: Step1-3の成果物が保存されていない

**解決方法**:
```bash
# ファイルの存在を確認
ls -la ideation/

# 必要なファイル:
# - step1_persona.md
# - step2_ideas.md
# - step3_prfaq.md
```

### エラー: Authentication failed

**原因**: `.env`ファイルが正しく設定されていない

**解決方法**:
```bash
# .envファイルの存在を確認
ls -la .env

# 内容を確認（APIキーが設定されているか）
cat .env

# 必要な設定:
# AWS_BEARER_TOKEN_BEDROCK=your-api-key-here
# AWS_REGION=ap-northeast-1
# AWS_BEDROCK_MODEL_ID=global.anthropic.claude-haiku-4-5-20251001-v1:0
```

### エラー: ModuleNotFoundError

**原因**: 仮想環境が有効化されていない

**解決方法**:
```bash
# 仮想環境を有効化
cd mock/
source .venv/bin/activate
cd ..
python presentation/generate.py
```

## 💡 Tips

### 美しいプレゼンのポイント

1. **スクロールを活用**
   - 各セクションがフルスクリーンで表示される
   - ゆっくりスクロールして説明

2. **アニメーションを活かす**
   - セクションが表示されるとアニメーションで登場
   - 自然な流れで観衆の注目を集める

3. **フルスクリーンモード**
   - F11キーでフルスクリーン
   - より没入感のある発表が可能

### 発表の流れ（3分）

1. **タイトル（10秒）** - サービス名を印象づける
2. **課題（30秒）** - 誰のどんな課題を解決するか
3. **解決策（60秒）** - 3つの機能を簡潔に説明
4. **デモ（40秒）** - 使い方のステップを説明
5. **強み（20秒）** - 独自性を強調
6. **チーム募集（20秒）** - 熱意を伝える
