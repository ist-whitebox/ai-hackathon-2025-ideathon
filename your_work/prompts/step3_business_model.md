# Step3: ビジネス化

## 目的
選択したアイディアをビジネスとして成立させるための分析を行う

## プロンプト

**前提**: Step2で選択した最有望案が記載されている `your_work/ideation/step2_ideas.md` を参照してください。

```
your_work/ideation/step2_ideas.md を参照してください。

選択したアイディアをビジネスとして成立させるための分析を行います。

1. ビジネスモデルキャンバス（オプション: 時間があれば）
   以下の順に記載してください：
   - 顧客セグメント（誰のための）
   - 価値提供（何を提供するか）
   - 販路（どうやって届けるか）
   - 顧客との関係（価格プラン）
   - 収益の流れ（どう稼ぐか）
   - 主要活動（何をするか）
   - 資源（何が必要か - 生成AIモデル、データなど）
   - 協力者（誰と組むか - OpenAI、AWSなど）
   - コスト構造（何にお金がかかるか）

2. ビジネスモデルの検証（オプション: 時間があれば）
   以下3点を確認し、表形式でまとめてください：
   ✓ 顧客の問題を解決しているか
   ✓ コストまたは収益の改善につながるか
   ✓ AIモデルの精度向上のためのデータ増加につながるか
   
   不十分な点があれば改善案を提示してください。

3. PR/FAQ（必須）
   以下を含めてください：
   - サービス名
   - 解決する課題（1文）
   - 体験手順（3ステップ）
   - 顧客FAQ
     - Q: どのように評価されていますか？
     - Q: 価格はいくらですか？
     - Q: 競合との違いは？
   - 提供者メッセージ
     - 市場規模
     - 目標（○ヶ月で○○達成）
     - 独自の強み

Markdown形式でまとめてください。

**重要**: 結果を `your_work/ideation/step3_prfaq.md` に保存してください。このファイルはモック作成時に参照されます。
```

## レビュー・修正フェーズ

生成されたPR/FAQを確認し、以下の観点で自分で考えて修正してください：

- [ ] サービス名は魅力的か？記憶に残るか？
- [ ] 解決する課題は明確か？伝わりやすいか？
- [ ] 体験手順はシンプルか？ユーザーが迷わないか？
- [ ] FAQの回答は納得感があるか？
- [ ] 独自の強みは明確か？競合と差別化できているか？

**必要に応じて `your_work/ideation/step3_prfaq.md` を直接編集してください。**

このPR/FAQがモック作成のベースになるので、特に重要です。

## 使い方

1. Step2で選択した最有望案を確認
2. 上記プロンプトをコピー
3. Amazon Q Developerチャットに貼り付け
   - VSCode: 左側のAmazon Qアイコンをクリック
   - （オプション）CLI: `q chat`
4. 出力結果を確認し、必要に応じて追加質問

## 重要

- **ビジネスモデルキャンバスはオプション**: 時間がない場合はスキップしてOK
- **PR/FAQは必須**: これがモック作成の元になります

## 期待されるアウトプット

- ビジネスモデルキャンバス（オプション）
- ビジネスモデルの検証（オプション）
- PR/FAQ（必須）
