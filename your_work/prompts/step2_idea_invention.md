# Step2: アイディア発明（10分）

## 目的
参考事例を収集し、組み合わせて新しい解決策を発明する

## プロンプト

```
先ほど特定した課題を解決するアイディアを発明します。

1. 参考事例の収集
   AWS生成AI事例集やWeb検索から、類似課題の解決事例を5つ探してください。
   各事例には以下を含めてください：
   - 概要
   - 技術的特徴（使用している生成AI技術）
   - ビジネス的特徴（収益モデル、顧客価値）

2. アイディアの発明
   収集した事例を2つ以上組み合わせて、新しい解決策を3つ発明してください。
   各アイディアには以下を含めてください：
   - アイディア名
   - 解決する課題（Step1の課題から選択）
   - 技術的アプローチ（どの生成AI技術を使うか）
   - 期待される効果（削減リソース、時間短縮など）
   - 実装難易度（低/中/高）

3. 最有望案の選択
   3つのアイディアから最も有望なものを1つ選び、理由を説明してください。
   選択基準：
   - 顧客インパクト（大きいほど良い）
   - 実装難易度（低いほど良い）
   - 独自性（競合との差別化）

表形式でまとめてください。

**重要**: 結果を `ideation/step2_ideas.md` に保存してください。
```

## 使い方

1. Step1の結果を確認
2. 上記プロンプトをコピー
3. Amazon Q Developer (`q chat`) に貼り付け
4. 出力結果を確認し、必要に応じて追加質問
